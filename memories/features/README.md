# memories/features/

このディレクトリは、新機能の追加や改修の目的と背景を記録するためのものです。

## 目的

新機能を追加するとき、既存機能を大きく改修するとき、APIやデータ構造を変更するときに、**実装の目的、画面構成、データ構造、完了条件**を記録します。

これにより、機能の意図や背景を後から振り返ることができます。

## 記録するケース

- 新機能を追加するとき
- 既存機能を大きく改修するとき
- APIやデータ構造を変更するとき

## ファイル名規則

```
YYYYMMDD-feature-name.md
```

例:
- `20251028-countdown-timer.md`
- `20251029-char-counter.md`

## テンプレート

```markdown
# [機能名]

## 日付
YYYY-MM-DD

## 実装目的
[なぜこの機能が必要か]

## 画面構成・UI
[どんな画面・UIか]

## データ構造
[使用するstate、propsなど]

## 実装ファイル
- src/...
- src/...

## リスク・注意点
[パフォーマンス、バンドルサイズなどの懸念]

## 完了条件
- [ ] 項目1
- [ ] 項目2
```

## 記録の原則

1. **機能単位**: 1機能 = 1ファイル
2. **目的を明確に**: なぜこの機能が必要か
3. **完了条件を設定**: 何ができれば完了か
4. **リスクを記載**: パフォーマンスやバンドルサイズへの影響
