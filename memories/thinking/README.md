# memories/thinking/

このディレクトリは、設計判断や思考過程を記録するためのものです。

## 目的

複数の実装方法で迷ったとき、技術選定で判断が必要だったとき、重要なアーキテクチャ上の決定をしたときなど、**なぜその判断をしたか**を記録します。

これにより、Claudeが過去の意思決定を理解し、一貫性のある開発を継続できます。

## 記録するケース

- 複数の実装方法で迷ったとき
- 技術選定で判断が必要だったとき
- パフォーマンスとシンプルさのトレードオフが発生したとき
- アーキテクチャ上の重要な決定をしたとき

## ファイル名規則

```
YYYYMMDD-short-description.md
```

例:
- `20251028-state-management-choice.md`
- `20251029-styling-approach.md`

## テンプレート

```markdown
# [判断内容の要約]

## 日付
YYYY-MM-DD

## 状況
[どんな問題・課題に直面したか]

## 検討した選択肢
1. [選択肢A]: メリット / デメリット
2. [選択肢B]: メリット / デメリット
3. [選択肢C]: メリット / デメリット

## 決定
[どの選択肢を選んだか]

## 理由
[なぜその選択をしたか]

## 懸念点
[将来見直す可能性があるか]
```

## 記録の原則

1. **簡潔に**: 1ファイル = 1トピック
2. **検索可能に**: ファイル名に日付と内容を含める
3. **結論ファースト**: 判断結果を先に書く
4. **未来の自分へ**: なぜその判断をしたかを明確に
