# 日付表示形式の切り替え機能

## 日付
2025-10-29

## 状況
現在の日付表示が「2025年/10月/29日（水）」となっており、「年月日」の漢字と「/」の区切り文字が混在している。日本語表記では、どちらか一方で統一した方が読みやすい。

## 検討した選択肢

### 1. 固定で「年月日」のみ（区切り文字なし）
- メリット: 日本語として自然、シンプル
- デメリット: ユーザーの好みに対応できない

### 2. 固定で「/」のみ（漢字なし）
- メリット: コンパクト、国際的
- デメリット: 日本語アプリとして違和感、ユーザーの好みに対応できない

### 3. 切り替え可能にする ✅
- メリット: ユーザーが好みで選択できる、柔軟性が高い
- デメリット: 設定項目が1つ増える（ただし最小限）

## 決定
**選択肢3: 切り替え可能にする**

ドロップダウンで以下の2形式を選択可能にする：
- **漢字表記**（デフォルト）: `2025年10月29日（水）`
- **スラッシュ区切り**: `2025/10/29（水）`

## 実装方針

### State追加
```typescript
type DateFormat = 'kanji' | 'slash'

interface ClockSettings {
  timezone: string
  layout: 'horizontal' | 'vertical'
  showSeconds: boolean
  dateFormat: DateFormat  // 新規追加、デフォルト: 'kanji'
}
```

### UI追加
- ドロップダウン（select）
- ラベル: 「日付の表示方法」
- オプション:
  - `漢字表記（年月日）`
  - `スラッシュ区切り（/）`
- 配置: レイアウト切り替えと秒数表示の間

### 表示ロジック
```typescript
{dateFormat === 'slash'
  ? `${year}/${month}/${day}（${weekday}）`
  : `${year}${month}${day}（${weekday}）`
}
```

## 理由
- 日本語表記では「年月日」が自然だが、「/」を好むユーザーもいる
- 設定項目を1つ追加するだけで、両方のニーズに対応できる
- 他の設定（秒数表示、レイアウト）と同様のパターンで統一感がある
- デフォルトを「区切り文字なし」とすることで、日本語として自然な表示を優先

## 懸念点
特になし。シンプルな機能追加で、既存の実装に影響を与えない。
