# memories/deleted/

このディレクトリは、削除・廃止した機能やファイルの履歴を記録するためのものです。

## 目的

ファイルを削除したとき、機能を廃止したとき、コンポーネントをリファクタリングで統合・削除したときに、**削除理由、影響範囲、代替手段、再発防止策**を記録します。

これにより、なぜ削除したかを後から振り返ることができ、同じ問題を繰り返さないようにします。

## 記録するケース

- ファイルを削除したとき
- 機能を廃止したとき
- コンポーネントをリファクタリングで統合・削除したとき

## ファイル名規則

```
YYYYMMDD-deleted-item-name.md
```

例:
- `20251028-deleted-old-timer.md`
- `20251029-removed-unused-component.md`

## テンプレート

```markdown
# [削除したもの]

## 日付
YYYY-MM-DD

## 削除内容
[何を削除したか]

## 削除理由
[なぜ削除したか]

## 影響範囲
[どのファイル・機能に影響があったか]

## 代替手段
[削除後、どう対応するか]

## 再発防止策
[同じ問題を起こさないために]
```

## 記録の原則

1. **削除前に記録**: 削除する前に記録を残す
2. **理由を明確に**: なぜ削除する必要があったか
3. **影響範囲を把握**: どこに影響があるか
4. **代替手段を示す**: 削除後はどうするか
